# 오픈채팅 수동 입력 UI 카피 가이드

## 1. 왜 자동 수집이 아닌 수동 입력인가

카카오 오픈채팅은 공식 API 를 통한 메시지 자동 수집 경로가 없습니다.
비공식 크롤링·스크래핑은 카카오 서비스 약관 위반이며,
**사용자 동의 없는 메시지 수집은 PIPA(개인정보보호법) 위반 소지**가 있습니다.

따라서 이 UI 는 운영자가 **직접 복사-붙여넣기**한 메시지만 처리합니다.
자동 수집 기능은 기술적으로 구현 가능하더라도 이 제품에서 제공하지 않습니다.

### 교육 자료 활용 포인트

> "왜 오픈채팅을 자동으로 못 가져오는지" 질문이 나오면
> 이 섹션을 직접 보여주세요. 약관·법률 이슈가 코드 설계에 반영된 실사례입니다.

## 2. 섹션 헤더 카피

| 위치 | 카피 | 비고 |
|------|------|------|
| 섹션 h2 | 오픈채팅 수동 입력 (정책상 자동 수집 X) | 괄호 안은 정책 이유 |
| 정책 badge | manual only · auto-collect blocked | 영문 소문자, mono 폰트 |

### 금지 표현
- "오픈채팅 크롤러" — 기능 암시
- "자동으로 불러오기" — 정책 위반 기능처럼 오해 가능
- "실시간 수집" — 동일

## 3. textarea placeholder

```
카카오톡 오픈채팅 메시지를 복사해서 붙여넣으세요.
한 메시지 = 한 줄 또는 빈 줄 구분.
```

### 파싱 규칙 (운영자 안내용)
- **빈 줄로 구분**: 긴 메시지 여러 개를 빈 줄로 구분하면 별개 건으로 분리됩니다.
- **한 줄 = 한 건**: 빈 줄이 없으면 줄 단위로 분리됩니다.
- 카카오톡 "내보내기" 텍스트 형식(날짜 헤더 포함)을 붙여넣어도 각 메시지만 추출됩니다.

## 4. 버튼 카피

| 상태 | 카피 | 색상 |
|------|------|------|
| 기본 | 정규화 + PMF Radar 진입 | `--accent` (#c8623a) |
| 처리 중 | 분류 중… | 비활성화 + `--muted` |
| 완료 후 결과 링크 | Radar 보기 ↓ | `--accent` 인라인 링크 |

## 5. 정책 안내 노트 카피

```
수동 입력만 허용 — 자동 수집은 약관·법률 이슈로 차단됩니다.
클라이언트에서 이메일·전화번호를 1차 가림 후 서버에서 최종 마스킹합니다.
운영자가 직접 붙여넣기한 메시지만 분석됩니다.
```

### 톤 원칙 (habix.ai writing-style-guide 기준)
- **사실 먼저**: 규제 이유를 먼저, 제약을 사과하지 않는다.
- **운영자 책임 명시**: 붙여넣기 행위는 운영자가 책임 주체임을 암시.
- **기술 용어 최소화**: PII, PIPA 는 필요할 때만. 일반 설명에서는 "이메일·전화번호" 로 풀어 씀.

---

## 6. 카운터 표시 카피

```
수동입력 N건 (오늘 누적 M건)
```

- `N`: 현재 textarea 파싱 건수 (실시간 갱신)
- `M`: localStorage `oc_today_count` / 날짜 기준 자동 리셋 (다음 날 0건 초기화)
- **만료 정책**: 날짜 변경 시 자동 초기화, Supabase 없이도 동작

---

## 7. PII 미리보기 카피

| 상태 | 표시 |
|------|------|
| 입력 전 | 텍스트를 입력하면 클라이언트 측 PII 1차 가림 결과가 여기에 표시됩니다. |
| 가림 없음 | N줄 파싱됨 · 1차 PII 가림 없음 |
| 가림 적용 | N줄 파싱됨 · 1차 PII 가림 적용됨 |

---

## 8. 오류 메시지 카피

| 상황 | 메시지 |
|------|--------|
| 빈 textarea | 붙여넣을 메시지가 없습니다. 텍스트를 입력해 주세요. |
| 서버 분류 실패 | 서버 연결 실패 — 로컬 기준으로 N건 분류 완료. |
| 분류 완료 | N건 분류 완료 · PMF Radar에 반영되었습니다. |

---

## 9. 운영자 책임 명시 원칙

이 UI 를 통해 메시지를 붙여넣는 운영자는 다음에 동의한 것으로 간주합니다:

1. **동의 기반 메시지만 사용**: 고객이 오픈채팅 공간에서 발화한 내용으로,
   별도의 개인 메시지(DM)가 아닌 공개 채팅 내용.
2. **PII 자체 점검**: 특이한 개인정보(주민번호, 계좌번호 등)는 붙여넣기 전 직접 제거.
3. **분석 목적 한정**: 이 도구에 입력된 메시지는 PMF 신호 분류에만 사용되며,
   외부 공개·마케팅 활용 금지.
